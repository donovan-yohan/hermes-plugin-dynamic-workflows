# v0.1.0 public release notes

`v0.1.0` starts semantic versioning for `hermes-plugin-dynamic-workflows`.

## Elevator pitch

Dynamic Workflows turns long-running agent work into explicit, resumable workflows: validate the plan,
run deterministic phases, park on durable waits, resume from external events, expose operator controls,
and clean up host-owned resources when the run exits.

## Intended users

- Hermes Agent users experimenting with plugin-hosted workflow orchestration.
- Agent engineers who need durable QA/review/release lanes rather than timer-driven watchdogs.
- Tool authors building host adapters for task boards, GitHub/webhooks, session launchers, or cleanup finalizers.
- Researchers comparing declarative agent workflows with script-led orchestration under a guarded capability boundary.

## Good fit examples

1. **PR validation lane**: webhook starts one workflow, QA/review run in parallel, exact-head evidence gates release recommendation.
2. **Issue lifecycle lane**: issue context seeds one run, implementation/review/QA phases advance through durable task events.
3. **Operator-approved repair loop**: sensors verify state, actuators request scoped grants for side effects, brakes halt runaway loops.
4. **Reusable script harness**: save a versioned Python orchestration harness and run it later with repo/issue/tool args.

## Not a good fit yet

- Running arbitrary untrusted code as a security boundary.
- Replacing a production workflow engine across multiple hosts without a shared event/run store adapter.
- Hiding credentials inside workflow inputs, task payloads, labels, or model prompts.
- Claiming a gateway/task-board integration works without a host adapter and a live smoke for that backend.

## Release checklist before creating GitHub Release

```bash
git status --short --branch
git rev-parse HEAD
uv run --extra dev pytest -q
uv build
python3 - <<'PY'
import glob, zipfile, tomllib
from pathlib import Path
version = tomllib.loads(Path('pyproject.toml').read_text())['project']['version']
assert version == '0.1.0', version
files = sorted(glob.glob('dist/*'))
print('version', version)
print('dist files', files)
assert any(f.endswith('.whl') for f in files), files
assert any(f.endswith('.tar.gz') for f in files), files
wheel = next(f for f in files if f.endswith('.whl'))
with zipfile.ZipFile(wheel) as zf:
    print('wheel entries', len(zf.namelist()))
PY
git diff --check
```

## Draft GitHub release notes

```text
# v0.1.0

First public alpha of Hermes Dynamic Workflows.

This release gives Hermes Agent a small, dependency-free workflow runtime for long-running agent work:
declarative workflow steps, guarded script harnesses, durable event waits, operator controls, scoped grants,
and resource finalizers.

Use it when an agent workflow needs to pause, wait, resume, retry, expose status, or clean up resources without
cron rediscovering the entire state every few minutes.

Alpha caveat: this is ready for local/plugin experimentation and adapter prototyping, not yet a hardened
multi-tenant sandbox for arbitrary untrusted users.

Artifacts:
- source archive
- Python wheel/sdist from `uv build`

Verification:
- `uv run --extra dev pytest -q`
- `uv build`
- wheel/sdist inspection
```
